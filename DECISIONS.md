# DECISIONS.md

## D001：第一版採用純前端靜態 PWA

- 日期：2026-07-19
- 決策：使用 HTML、CSS、JavaScript 建立靜態 PWA。
- 原因：符合第一版需求，降低 iOS / iPadOS Safari 相容性風險，並可透過 Service Worker 支援離線使用。

## D002：聲音播放採真人錄音優先、語音合成備援

- 日期：2026-07-19
- 決策：JavaScript 先尋找 `assets/audio/*.mp3` 真人錄音；載入失敗或未提供時，改用 Web Speech API。
- 原因：第一版可快速驗證互動流程，同時保留未來替換成真人錄音的機制。

## D003：學習紀錄僅儲存在本機

- 日期：2026-07-19
- 決策：使用 `localStorage` 保存星星數、答題數與最後練習時間。
- 原因：符合不收集兒童個人資料的原則，也能支援離線學習。

## D004：家長模式第一版不設密碼

- 日期：2026-07-19
- 決策：第一版提供家長模式入口、隱私說明與本機紀錄重置，不加入帳號或密碼。
- 原因：避免收集資料與降低導入門檻；未來可再增加家長確認題。

## D005：音訊狀態改由 manifest 統一管理

- 日期：2026-07-19
- 決策：新增 `assets/audio/audio-manifest.json`，只有 `status=recorded` 時才視為真人錄音。
- 原因：避免缺檔時誤導使用者，也方便未來逐步補齊真人錄音。

## D006：瀏覽器語音合成只唸例字並標示限制

- 日期：2026-07-19
- 決策：缺少真人錄音時使用 Web Speech API 唸例字作備援，不宣稱為標準單一注音發音。
- 原因：不同瀏覽器與系統語音不一定能正確發出單一注音符號。

## D007：本次提交暫時只引用文字格式 SVG icon

- 日期：2026-07-19
- 決策：本次為確保 GitHub 推送相容性，暫時移除 PNG icon 引用，只保留文字格式 SVG icon；192×192、512×512 與 Apple 180×180 PNG icon 改列下一階段待辦。
- 原因：目前推送環境不支援二進位檔案，先避免 manifest、HTML 與 Service Worker 引用不存在的 PNG。
